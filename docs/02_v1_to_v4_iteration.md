# V1 to V4 Iteration

## V1 — IV Voiceprint Smart Contract
Focus:
- Store and validate 7-value IV voiceprint data.
- Oracle-signed reports.
- Sepolia smart contract prototype.

Main value:
- Proves that IV-derived data can be structured, signed, validated, and recorded.

Limitation:
- Validation rules are still heuristic.
- It does not yet establish full PUF characterization.

## V2 — DSSC PUF / Physical Co-Factor
Focus:
- Treat patterned DSSC as a physical co-factor.
- DSSC is not a replacement for TPM or secure element.
- DSSC provides material response evidence.

Main value:
- Shifts the project from pure blockchain storage to physical-material identity.

Limitation:
- Needs characterization, repeatability testing, challenge-response design, and environmental compensation.

## V3 — Material Event Signature
Focus:
- Move from static identity to event-based evidence.
- A measurement is not only "who is this device?" but "did this physical material event happen under this challenge?"

Main value:
- Better language for energy provenance and art-tech material practice.

Limitation:
- Requires stronger event schema, raw data hashing, and reproducible measurement protocols.

## V4 — Public Research Registry / Energy Provenance Evidence
Focus:
- Public GitHub-based registry.
- Device metadata.
- Measurement event records.
- Daily hashes.
- Optional blockchain anchoring.
- Optional LLM explainer.

Main value:
- Makes the project understandable and reviewable by advisors, investors, and the public.

Limitation:
- GitHub registry is public and versioned, but not itself a decentralized immutable ledger.

This evolution is a research framing, not a claim that V4 is fully implemented.
